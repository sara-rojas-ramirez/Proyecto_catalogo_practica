# IMPORTACIONS BASICAS USADAS
1. render y redirect: Se usan para mostrar plantillas HTML o redirigir al usuario a otra URL.
2. authenticate, login y logout: Son funciones nativas de Django que sirven para verificar si la contraseña es correcta, crear sesión y destruirla. 
3. Loginform: Un formulario personalizado (que se crea en un form.py) para pedir username y password.



# MIGRACIONES
Solo se realizan migraciones cuando se cambia la estructura del modelo. Se realiza makemigrations y migrate.
Cuando se hacen registros, no es necesario migrar.


# ESTRUCTURA DEL PROYECTO (Separación de responsabilidades)
---> accounts: Se encargara del login, logout, registro, auntenticación y roles.

---> clientes: Para guardar información extra del cliente. Donde user tiene una relación 1 a 1 con cliente

---> catalogo: Se encarga de administrar productos, agrupar los productos por medio de categoria y mostrar el proveedor. 

---> carrito: Contiene el carrito para guardar temporalmente los productos antes de comprar, con item_carrito para mostrar info de cada producto y las operaciones CRUD basicas. 

---> checkout: Se encarga de mostrar resumen, formulario de envio, simular pago y confirmación de pago exitoso. 

---> pedido: Pedidos ya creados, entonces, tiene los modelos pedido y detalle_pedido, las vistas de historial del cliente y detalle_pedido. 

---> static: Aqui van los archivos fijos del diseño. Como por ejemplo, el logo, donde de la pagina, styles o iconos. 

---> media: En esta carpeta es donde se guardan automaticamente las imagenes que el usuario o admin en este caso, suba por medio del modelo de Producto, a traves de upload_to = productos/
